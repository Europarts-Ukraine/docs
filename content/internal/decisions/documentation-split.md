# Documentation Split

The documentation system has two build targets.

## Public Documentation

Public documentation is the customer-facing Help Center and optional integration reference. It must not expose source-code, deployment, or internal architecture details.

## Internal Documentation

Internal documentation includes customer docs plus implementation, support, engineering, architecture, operations, and decision records.

## Repository Model

Both documentation targets live in this repository so documentation can evolve together with code. Deployment decides which target is published.
